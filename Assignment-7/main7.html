Normalization

Normalization is the process of breaking down tables into normal forms to reduce data redundancy and improve data integrity.

Types of Normalization

  • 1NF (First Normal Form)

    Each column contains atomic (indivisible) values. Basically there should not be any group or collection of data inside an entity. Example:

    
  
    IDNamePhone
    
  
    1John1234567890

  • 2NF (Second Normal Form)

    Table is in 1NF, and all non-key attributes are fully functional dependent on the primary key. Here, it means to say that the every entity or column should be related to the primary key of the table. In each table the columns should be directly dependent on primary key of the original table. Example:

    
  
    IDNameCourse
    
  
    1JohnMath

  • 3NF (Third Normal Form)

    Table is in 2NF, Here it means to say that there should not be any transitive dependencies like table1 is related to Table2 and Table 2 is related to Table 3 the Table 3 should not directly be related with Table 1. Example:

    
  
    IDNameCourseInstructor
    
  
    1JohnMathMr. Smith
